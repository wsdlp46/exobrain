> 外脑不是预设行业知识的方法论库，也不是静态文件的归档柜。它是一个工作记忆与知识系统实时耦合的自进化引擎。你扔进去的每一份材料——文档、会议纪要、方案、决策——都在被自动拆解、索引、关联成动态知识网络。下一次你需要写任何内容时，外脑自动调取所有相关的历史经验、相似场景和踩坑记录作为上下文，让你不必从零开始。知识库不是背景板，而是工作流的前置引擎；工作流不是机械流程，而是知识网络持续生长的管道。两者互为因果，螺旋上升。它不是替你检索的工具，是外置一个随你工作而持续校准认知方式的第二大脑。

## 这是什么？

**外脑（Exobrain）** 不是预设行业知识的方法论库，也不是静态文件的归档柜。它是一个**工作记忆与知识系统实时耦合的自进化引擎**。你扔进去的每一份材料——文档、会议纪要、方案、决策——都在被自动拆解、索引、关联成动态知识网络。下一次你需要写任何内容时，外脑自动调取所有相关的历史经验、相似场景和踩坑记录作为上下文，让你不必从零开始。你工作得越多，知识网络越密，上下文加载越精准，它对你工作方式的适配越自然。知识库不是背景板，而是工作流的前置引擎；工作流不是机械流程，而是知识网络持续生长的管道。两者互为因果，螺旋上升。零预设、全自适应，你喂什么，它学什么；你用它，它懂你。本质上，这是**外置一个随你工作而持续校准认知方式的第二大脑**。

它是一个通用 AI 工作搭档，面向所有知识工作者——设计师、运营、咨询、开发、管理者，不限行业。它把你的工作记忆外置，让 AI 帮你存取每一次的经验、决策和踩坑记录。

它和 [pm-exobrain](https://github.com/wsdlp46/pm-exobrain) 同属 [WorkBuddy AI 协作体系](https://github.com/wsdlp46)，定位完全不同：

| 维度 | pm-exobrain | 外脑（Exobrain） |
|------|------------------|-----------------|
| **用户** | 产品经理 | 设计师 / 运营 / 咨询 / 开发 / 管理……所有知识工作者 |
| **角色认知** | "我是 10 年 PM，擅长 PRD/方案/原型" | "我是你的外脑，帮你记经验、找案例、避踩坑" |
| **内置知识** | PM 方法论、案例库、术语、B 端产品模式 | **零预设**——无任何行业知识、角色认知、方法论 |
| **工作流** | PRD 撰写 / 方案设计 / 原型设计 | 搭结构 → 入库 → 加载上下文 → 复盘回流 → 自进化 |
| **上手方式** | 开箱即用，直接写 PRD | **先扔材料，再开始用**——所有知识源于你的材料 |

## 核心能力

### 搭结构
装好后扔进第一批材料，外脑自动识别你的文件类型和工作模式，推荐最适合的目录结构——项目制、持续运营、混合模式或个人知识库。

### 消化材料
调用 project-intake Skill，自动分类归档：哪些进知识库、哪些进案例库、哪些只是过程记录。

### 辅助干活
开始干活时自动加载相关上下文：关键决策、历史案例、设计规范、踩坑记录——每次干活不必从零开始。

### 复盘回流
调用 project-retrospective Skill，扫描过程、归因修复、更新知识库——每次项目都让知识库更强。

### 自进化
每完成一个项目，外脑更了解你的工作模式，下次推荐更准。

## 快速开始

### 安装

在 WorkBuddy 桌面客户端中，左侧边栏「专家」→ 右上角「+」→「从本地文件导入」→ 导入本项目目录。

### 第一步：准备一个空目录

在你的电脑上创建一个空目录，作为工作区根目录。例如：

```
/Users/xxx/我的工作区/
```

这个目录是**你的**知识库，所有材料都存在这里，外脑只负责读取和写入，不会把数据存到别处。

### 第二步：扔材料

把你手头的工作材料（至少 3 份）放进这个目录，可以是：
- 文档（.md、.docx、.pdf）
- 会议纪要（任何格式）
- 方案/报告
- 设计稿（.figma、.sketch、图片）

然后对外脑说：

> 帮我搭一套知识库结构，我的工作区根目录是 `/Users/xxx/我的工作区`，这是我的第一批材料

外脑会：
1. 分析你的材料类型（是项目制？持续运营？个人笔记？）
2. 推荐最合适的目录结构（4 种模板之一）
3. 向你展示推荐方案和理由
4. 你确认后，创建目录结构并生成 `_index.md`

### 第三步：入库

结构搭好后，外脑会逐批把你的材料分类入库：
- 关键决策 → 提炼到 `关键决策.md`
- 会议时间线 → 整理到 `会议纪要_时间线.md`
- 过程记录 → 归档到 `过程记录/`
- 可复用案例 → 进入 `03知识库/案例库/`

> **知识库存哪？** 全部存在你的工作区目录里，是 Markdown 文件，不锁在任何工具里。你随时可以用任何编辑器打开。

### 第四步：开始干活

以后每次要开始工作，对外脑说：

> 开始干活，帮我加载相关上下文。我要写 [具体任务]

外脑会从你的知识库中自动匹配：
- 同类项目的历史决策
- 相关案例和经验
- 已知的踩坑记录
- 设计规范或标准

### 第五步：定期复盘

项目阶段性完成后，对外脑说：

> 复盘这个项目，更新知识库

外脑会扫描整个项目过程，找出可改进的地方，自动更新知识库——案例更厚、模式更准、坑更少。

## 一个完整的工作周期

以下是一个设计师使用外脑的完整示例：

**第 1 天 — 搭建**

> 你：帮我搭一套知识库结构，我的工作区是 `/Users/xxx/设计工作区`，材料是这些：[上传 5 份过往设计稿和评审纪要]
>
> 外脑：分析你的材料，主要是 UI 设计稿和评审反馈，推荐「项目制」目录结构。确认后创建目录。

**第 2 天 — 入库新项目材料**

> 你：入库这个文件：`/Users/xxx/设计工作区/新电商项目/需求文档.md`
>
> 外脑：提取关键决策，更新 `_index.md` 和 `关键决策.md`，归档到 `01-项目输入/`。

**第 3 天 — 开始干活**

> 你：开始干活，帮我加载上下文。我要给新电商项目做首页设计。
>
> 外脑：加载知识库中所有电商相关案例和评审反馈，作为上下文。开始设计。

**第 10 天 — 项目结束，复盘**

> 你：复盘这个电商项目，更新知识库。
>
> 外脑：扫描设计过程，发现 3 个可复用的设计模式、2 个新踩坑点，更新案例库和错误笔记。

**第 11 天起 — 越用越聪明**

下一次你做电商项目时，外脑已经知道你的设计偏好、踩过的坑、评审标准，上下文加载更精准。

> **注意**：exobrain 本身只带工作流引擎，不存储你的知识。你的知识全部存在自己的工作区目录里，外脑只是帮你组织、索引和调用。

## 设计理念

**文件即知识库。** 所有知识存 Markdown 文件，不锁在任何工具里。目录结构本身就是导航系统。

**零预设 = 真通用。** 不假设你是哪个岗位、什么行业。知识全部来自你的材料：
- 设计师用，它学设计规范和评审模式
- 开发者用，它学技术选型和架构决策
- 咨询顾问用，它学方法论和交付标准
- 运营用，它学活动复盘和数据看板

**渐进式理解。** 第一个项目周期只入库不推荐，第二个周期才开始主动匹配。越用越懂你。
