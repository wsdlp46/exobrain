# 自适应目录模板

根据用户的工作模式，推荐不同的目录结构。以下模板均为推荐起点，用户可自定义调整。

---

## 模板 1：项目制（project-based）

**适用场景：** 工作以独立项目为单位，有明确的起止时间和交付物。如设计项目、咨询项目、工程实施项目。

```
项目根目录/
├── _index.md                  ← 全局索引（项目列表、状态、负责人）
├── 00_通用/                   ← 跨项目共享
│   ├── 工作流规范.md
│   ├── 模板库/
│   │   ├── 方案模板.md
│   │   └── 报告模板.md
│   └── 术语表.md
├── 01_进行中/
│   ├── 项目A/
│   │   ├── _项目概览.md       ← 项目背景、目标、关键决策索引
│   │   ├── 01_输入/           ← 需求文档、会议纪要、参考资料
│   │   ├── 02_过程/           ← 中间版本、工作记录
│   │   ├── 03_产出/           ← 最终交付物
│   │   └── 04_复盘/           ← 项目复盘报告
│   └── 项目B/
│       └── ...
├── 02_已完成/
│   └── 项目X/                 ← 已完成项目归档（仅保留最终产出和复盘）
├── 03_案例库/
│   ├── 案例库主页.md          ← 按领域/类型索引
│   └── 案例_XXX.md            ← 单案例详情
├── 04_参考资料/
│   ├── 行业报告/
│   └── 竞品分析/
└── 05_关键决策.md             ← 跨项目关键决策汇总
```

**推荐理由：** 项目制工作的知识复用主要发生在"相似项目"之间，因此按项目分组，案例库跨项目索引。

---

## 模板 2：持续运营（ops-based）

**适用场景：** 工作围绕持续运营的业务线/产品线展开，无明确结束时间。如产品运营、内容运营、平台运维。

```
项目根目录/
├── _index.md                  ← 全局索引
├── 00_通用/
│   ├── 工作流规范.md
│   ├── 设计规范.md             ← 持续维护的规范文件
│   ├── 检查清单.md
│   └── 术语表.md
├── 01_业务域A/
│   ├── _域概览.md              ← 业务背景、核心指标、团队
│   ├── 关键决策.md             ← 该域的关键决策时间线
│   ├── 方案/                   ← 历次方案文档
│   ├── 数据/                   ← 数据报告、看板截图
│   ├── 复盘/                   ← 周期性复盘（季度/年度）
│   └── 会议纪要_时间线.md
├── 02_业务域B/
│   └── ...
├── 03_案例库/
│   └── 案例库主页.md
└── 04_参考资料/
    └── ...
```

**推荐理由：** 持续运营的知识沉淀按业务域组织，时间线是关键检索维度；规范文件持续更新而非版本锁定。

---

## 模板 3：混合模式（hybrid）

**适用场景：** 同时有项目制工作和持续运营任务。如设计团队（项目制交付 + 设计规范维护）、咨询公司（项目交付 + 方法论沉淀）。

```
项目根目录/
├── _index.md
├── 00_通用/                   ← 跨项目和运营共享
│   ├── 工作流规范.md
│   ├── 模板库/
│   └── 术语表.md
├── 01_项目/                   ← 项目制工作
│   ├── 进行中/
│   │   └── 项目A/
│   └── 已完成/
├── 02_运营/                   ← 持续运营工作
│   ├── 业务域A/
│   └── 业务域B/
├── 03_案例库/
├── 04_参考资料/
└── 05_关键决策.md
```

**推荐理由：** 项目制工作和持续运营工作的知识流转方式不同——项目产出是案例，运营产出是持续更新的规范。分开组织可避免混淆。

---

## 模板 4：个人知识库（personal-kb）

**适用场景：** 个人学习笔记、阅读摘录、方法论整理，不涉及团队协作。

```
个人知识库/
├── _index.md                  ← 主题索引
├── 01_工作/
│   ├── 项目经验/
│   ├── 踩坑记录.md
│   └── 工具链.md
├── 02_学习/
│   ├── 读书笔记/
│   ├── 课程笔记/
│   └── 技术文档/
├── 03_写作/
│   ├── 文章草稿/
│   └── 已发布/
└── 04_素材/
    ├── 图片/
    └── 模板/
```

**推荐理由：** 个人知识库的核心是"快速找到"而非"团队协作"，目录按主题而非项目组织，索引是第一优先级。

---

## 推荐决策流程

```
用户提供第一批材料
       │
       ▼
扫描文件类型和内容特征
       │
       ├─ 有明显的项目文件夹 + 起止时间 → 模板1（项目制）
       ├─ 按业务线/功能模块组织 + 时间线为主轴 → 模板2（持续运营）
       ├─ 两者兼有 → 模板3（混合模式）
       └─ 以个人笔记/学习材料为主 → 模板4（个人知识库）
       │
       ▼
向用户展示推荐方案和理由
       │
       ▼
用户确认后创建目录 + _index.md
```

## 索引文件格式（_index.md）

无论选择哪个模板，根目录的 `_index.md` 统一采用以下格式：

```markdown
# [项目/知识库名称]

## 基本信息
- 工作模式：[项目制/持续运营/混合/个人]
- 创建时间：YYYY-MM-DD
- 最近更新：YYYY-MM-DD

## 目录导航
- [00_通用]() — 工作规范、模板、术语
- [01_项目]() — 项目制工作
- [02_运营]() — 持续运营工作
- [03_案例库]() — 历史案例索引
- [04_参考资料]() — 外部资料
- [05_关键决策]() — 关键决策汇总

## 最近入库
| 时间 | 文件 | 分类 |
|------|------|------|
| YYYY-MM-DD | 文件名 | 类型 |

## 高频标签
#标签1 #标签2 #标签3
```
